⚙️ How It Works

From your first click to a personalized recommendation in milliseconds

Step 1
🔍

You Search or Browse

Search for any movie or browse trending picks. New users get top-rated movies powered by a Bayesian popularity engine.

Step 2
🧠

AI Models Activate

Depending on context, the system routes your request to the best-suited model — ALS, NeuMF, Content-Based, or Popularity.

Step 3
🎨

Enrichment Engine

10 parallel threads fetch movie posters from OMDB and trailer IDs from YouTube for every result — in real-time.

Step 4
⭐

You Rate, It Learns

Every star rating you submit is saved to Supabase. The NCF model fine-tunes itself on your new data in under 5 seconds.

🤖 The 4 AI Models

Each model solves a different part of the recommendation problem

⚡

ALS — Implicit Collaborative Filtering

Matrix Factorization

Decomposes the 32M rating matrix into 50-dimensional user and movie embeddings using Alternating Least Squares. Uses all CPU cores via C++ OpenBLAS threading.

Speed: ~20ms per query · Factors: 50 · Iterations: 15
🧠

NeuMF — Neural Collaborative Filtering

Deep Learning · PyTorch

Hybrid neural network combining GMF (linear) + MLP (non-linear) towers. Content-aware via genre one-hot encoding + tag EmbeddingBag. Fine-tunes on live data in ~5 seconds.

Architecture: 128→64→32 · Dropout: 0.2 · Optimizer: Adam
🎭

Content-Based Filtering

TF-IDF + Cosine Similarity

Builds a feature matrix from movie genres and 1.1M+ user-generated tags via TF-IDF vectorization. Finds movies similar in style without needing any user data.

Features: 5,000 TF-IDF · Similarity: Chunked L2 cosine
🔥

Popularity-Based Model

Cold Start Solution

Uses the IMDb-style Bayesian weighted rating formula to rank movies. Prevents low-vote movies from ranking above well-established favorites. Perfect for new users.

Formula: Weighted Bayesian Average · Threshold: 70th percentile
